## 1. 研究背景と動機

### 1.1 従来の課題

光学顕微鏡で透過物体（細胞・組織等）の3次元構造を得るには、焦点位置を変えながら複数枚の**多焦点画像**を逐次撮影する必要がある。

| 課題 | 内容 |
|------|------|
| **データ量の増大** | D枚の画像が必要（例：D=11枚、約176kB） |
| **撮影時間の増加** | 焦点位置を1回ごとに移動させながら逐次撮影 |
| **適用限界** | 高速に運動する細胞の観察や大量サンプルのスクリーニングに不向き |

### 1.2 提案のアイデア：ビデオCSの概念を深度軸へ拡張

**ビデオ圧縮センシング（ビデオCS）** は、時間軸(t)の情報を画素別シャッタ符号化によって1枚の画像に圧縮し、高フレームレート動画を復元する技術。

提案手法は、この「**時間軸(t)の符号化**」という概念を「**深度軸(z)の符号化**」へ拡張する。

| 概念 | ビデオCS | 提案手法 |
|------|----------|----------|
| 圧縮する次元 | 時間軸 t | 深度軸 z |
| 圧縮メカニズム | 画素別シャッタの時間符号化 | 画素別シャッタの焦点深度符号化 |
| 撮影 | 1枚の圧縮フレーム | 1枚の圧縮多焦点画像 |
| 復元対象 | 高フレームレート動画 | 3次元透過率構造 |
| 物理モデル | 電子シャッタの積算 | PSF + シャッタの積算 |
| 正則化 | CNN（RevSCI等）/ DIP | DIP（3D U-Net） |

**メリット：**
- 入力データ量の大幅削減（卒論実績：約91%削減、11枚→1枚）
- 撮影時間の短縮（1フレームで全深度情報を取得）

## 4. 第1段階（卒業論文）詳細

### 4.1 システム概要

提案手法は2つの要素で構成される（図1参照）：

```
[圧縮画像撮影]                    [3次元復元]
焦点可変レンズ                    ランダムノイズ
     +          →  圧縮画像  →        ↓
画素別シャッタ制御センサ           3D U-Net
                                      ↓
                              推定3次元構造
```

### 4.2 ハードウェア

#### 卒論で想定したカメラ：MAUF19

| 仕様 | 内容 |
|------|------|
| 制御単位 | 近傍2×2の4画素（シャッタ制御ブロック） |
| サブ画素 | 1画素に4サブ画素（うち1つはドレイン、3つがデータ蓄積） |
| パターン数 | 12個（3タップ×4画素）、うち11個を11深度に割り当て |
| 特性 | 各画素に独立したシャッタ制御、数100MHz以上の動作速度を想定 |

MAUF19は高周波数（数100MHz以上）を想定しており、TAGレンズ（70kHz）の低周波数に合わせることができなかったため、**現在は新カメラへ変更**。

#### 現在使用予定の新カメラ

| 仕様 | 内容 |
|------|------|
| 制御単位 | 2×2ピクセルブロック |
| タップ | 1画素に2タップ + ドレイン |
| 当面の使用 | **1タップのみ使用**（各画素はオン/オフの2値制御） |
| 有効パターン数 | 2^4 = 16通り（全閉・全開除くと**14通り**） |
| フレーム内制約 | **同一フレーム内は全ブロックが同一パターン（タイリング）** |
| 最小時間窓幅 | 500ns程度 |
| 対応レンズ | TAGレンズ（70kHz）に対応可能 |

> **補足（将来的な拡張）：**  
> 2タップを使用すると、タップAとタップBで異なるパターンを露光できるため、**1フレームで2枚の圧縮画像**を同時に取得可能。現状は1タップのみ使用。

#### ハードウェア制約とシミュレーションの対応

「同一フレーム内は同一パターン（タイリング）」という制約は、カメラの物理仕様によるものであり、コード内のブロックタイリング実装はこの制約を正しく再現している（理論的なCSの問題ではなく、意図的な設計）。

### 4.3 順方向モデル（圧縮イメージングモデル Φ）

2段階のパイプライン：

#### Stage 1：イメージングモデル（物理光学、Yamaguchi準拠）

3次元構造 α から各焦点位置での多焦点画像 F̂_z(x,y) を生成。

**光線減衰モデル：**

$$l'_j = l_j \prod_i \alpha_i^{d_{ji}}$$

- l_j：j番目の光線の初期強度
- α_i：ボクセルiの透過率
- d_{ji}：光線jがボクセルi内を移動する距離

**画素強度：**

$$\hat{F}_z(x, y) = \sum_j l'_j$$

#### Stage 2：圧縮モデル（シャッタ符号化）

多焦点画像にシャッタパターンを適用し、深度方向に積算して1枚の圧縮画像を生成。

$$\hat{I}(x, y) = \sum_{z=1}^{D} \hat{F}_z(x, y) \cdot S_z(x, y)$$

- S_z(x,y) ∈ {0,1}：深度zに対応するシャッタパターン
- D：深度スライス数（卒論では D=11）

**順方向モデル全体：** `Î = Φ(α, S)`

> 注：コード上では `ImagingModel`（Azevedo準拠）と `MaskedImagingModel`（提案手法）が実装されている（`img_model.py`）。

### 4.4 3次元復元（DIP）

#### アーキテクチャ

```
固定ランダムノイズ n ∈ R^{H×W×D}
        ↓
    3D U-Net (fθ)
        ↓
 推定3次元構造 α̂ = fθ(n)
        ↓
  圧縮イメージングモデル Φ
        ↓
シミュレーション圧縮画像 Î
        ↓
  L1 Loss と実観測画像 I を比較
        ↓
   θ をバックプロパゲーションで更新
```

- 入力ノイズ n は最適化中**固定**
- パラメータ θ のみを更新

#### 損失関数

$$\mathcal{L}(\theta) = \|I - \hat{I}\|_1$$

#### 事前学習（Pretraining）

楕円球（細胞の形状を近似）にガウスノイズを加えたペアデータで教師ありデノイジングを実施。

- L2 Loss、500イテレーション
- 目的：収束高速化 + 復元精度向上
- Azevedo et al.のPretr II相当

### 4.6 評価指標

| 指標 | 説明 | 方向 |
|------|------|------|
| **RMSE** | ボクセルごとの二乗平均平方根誤差 | ↓ 低いほど良い |
| **Dice Score** | 3次元構造の形状類似度 | ↑ 高いほど良い |
| **SSIM** | 輝度・コントラスト・構造の類似性 | ↑ 高いほど良い |
| ~~PSNR~~ | ~~ピーク信号対雑音比~~ | **結果が良くないため非公開** |

### 4.7 実験結果

#### 合成データ（定量評価）

| 手法 | 入力 | 平均RMSE↓ | 平均Dice↑ | 平均SSIM↑ |
|------|------|-----------|-----------|-----------|
| **従来手法**（多焦点画像） | 11枚（176kB） | 0.0070 | 0.9679 | 0.9896 |
| **提案手法**（圧縮画像） | 1枚（16kB） | 0.0125 | 0.9664 | 0.9799 |

**主要サンプルの結果：**

| データセット | RMSE（従来） | RMSE（提案） | Dice（従来） | Dice（提案） |
|------------|-------------|-------------|-------------|-------------|
| aneurism | 0.0035 | 0.0117 | 0.9945 | 0.9944 |
| beechnut | 0.0048 | 0.0070 | 0.9788 | 0.9799 |
| chameleon | 0.0070 | 0.0135 | 0.9532 | 0.9574 |
| pancreas | 0.0049 | 0.0093 | 0.9325 | 0.9323 |

**入力データ量：約91%削減**（176kB → 16kB）

#### 実データ（定性評価）

- 細胞の境界を明瞭に捉え、生体組織特有の滑らかな曲面構造を高忠実度で再現
- DIPの強力な正則化効果が実データでも有効に機能

### 4.8 考察と限界

#### 成果

- データ量91%削減にもかかわらず、Dice Scoreはほぼ同等（0.9679→0.9664）
- 単一圧縮画像からの3D復元の実現可能性を実証
- ビデオCSの概念を深度軸へ拡張する新しい枠組みの提案

#### 課題・限界

| 課題 | 原因 | 詳細 |
|------|------|------|
| RMSEの悪化 | 「重なり」の影響 | 単一画素に複数深度情報が混在し、完全に分離できていない |
| PSNRが非公開 | 数値が低い | 定性的には良好だが定量的に課題あり |
| 実撮影なし | シミュレーションのみ | 実際のカメラによる撮影・検証は未実施 |
| パターン未最適化 | ランダムパターン使用 | 第2段階で解決予定 |

## 5. 第2段階（修士論文）

### 5.1 目的

シャッタパターンの最適化により、復元精度（特にRMSE）をさらに向上させる。

### 5.2 新カメラの制約（再掲）

- 2×2ブロック単位でパターン決定
- 1タップのみ使用（各画素オン/オフの2値）
- **有効パターン数：14通り**（全閉・全開除く）
- 同一フレーム内は同一パターン（タイリング）

### 5.3 想定アプローチ：Deep Sensing（勾配法）

**Yoshida et al. (Sensors 2023)** の手法を参考に：

```
[センサ層]                        [復元層]
シャッタパターン φ（バイナリ）         ↓
  ↓ 順方向（バイナリ重みを使用）    DIPまたは復元ネットワーク
  圧縮画像 y = Φ(x, φ)              ↓
  ↓ 逆方向（連続重みに切替）        推定3次元構造 α̂
  勾配を計算                         ↓
  ↓                              Loss
  連続重みを更新
  ↓
  バイナリ重みを更新（閾値処理）
```

**ポイント：バイナリ制約の扱い**
- 順伝播：バイナリ重みを使用（実際のハードウェアを模倣）
- 逆伝播：連続重みに切り替えて勾配計算
- 更新後：閾値処理でバイナリ重みに変換

**ハードウェア制約の組み込み：**
- 2×2ブロックタイリングをネットワーク構造に反映
- 全閉・全開パターンを禁止する制約を学習中に考慮

### 5.4 解決すべき課題

| 課題 | 内容 |
|------|------|
| バイナリ最適化 | 勾配が計算できない2値変数の扱い |
| 学習データ | DIPベースで教師データ不要だが、パターン最適化のための訓練データが必要 |
| 評価設計 | どの指標でパターンを最適化するか（RMSE？Dice？） |
| ハードウェア検証 | 最終的には実際の新カメラで検証が必要 |

## 8. キーワード・用語集

| 用語 | 説明 |
|------|------|
| 多焦点画像 | 焦点位置を変えながら撮影した複数枚の画像スタック |
| 圧縮多焦点画像（圧縮画像） | 全深度情報を1枚に符号化した画像 |
| シャッタパターン S_z(x,y) | 深度zにおける各画素の開閉状態（{0,1}） |
| Deep Image Prior (DIP) | 教師データ不要の逆問題解法、CNNの構造的バイアスを正則化として利用 |
| 圧縮センシング (CS) | 信号のスパース性を利用した少ない観測からの復元理論 |
| ビデオCS | 時間軸をシャッタ符号化し1枚から動画を復元 |
| タイリング | 同一ブロックパターンを画像全体に繰り返し適用（ハードウェア制約） |
| RMSE | Root Mean Square Error（ボクセル誤差） |
| Dice Score | 3次元構造の形状類似度（集合の重複率） |
| Deep Sensing | センシング行列と復元デコーダを同時にエンドツーエンドで最適化する手法 |
